Doing Stuff with Web Things (Local Storage)

"REMEMBERING USER'S INFO

  • 1. How can I store data so it isn't lost when the application stops?
  • 2. How do I get data back into my application once it has been stored?
  • 3. What can I do to allow the user to communicate?

-COMMUNICATION MUST BE BI-DIRECTIONAL ---> THE USER NEEDS SOME WAY TO SPEAK TO YOU.

-COMPUTERS ARE STUPID -> THEY CAN'T REMEMBER ANYTHING, UNLESS THE APPLICATION IS DESIGNED TO STORE WHAT IT HAS DONE.

*Reminder* OUTPUT is the information stored or shared with the USER. INPUT is anything coming in from where it was stored or anything the USER enters.


***Javascript Functions***

  • Function goes in the " { } " characters... " LET " or " CONST " tells the computer to create a variable.
  • When you want to get the user's input from any input or text area tag: you can use the special word "VALUE"
  • When you want to show something to the user: USE A "SECTION" and "INNERHTML"
  • ---- Don't get them mixed up.
  • ------------ INPUT/TEXTAREA = VALUE
  • ------------ SECTION = InnerHTML
  • Using JS you can store any information you want to use later. ---- even if user refreshes page or goes to another site & comes back.
  • ----- Using "localStorage" (to save content)
  • ------- JS uses 'Key-Value' pairs. An example is KEY= TITLE OF MUSIC ----- VALUE= THE AUDIO FILE
  • **Code Example**
  • - function loadStory(){
  • --- let storyName = document.getElementById ("name_input").value
  • --- let storyHtml = localStorage.getItem(storyName)
  • --- document.getElementById("story_editor").value = storyHtml

SITEPOINT "MASTER MOBILE UX"

-Lesson 1, Step 1----How to rewire your brain for mobile first UX


Joel Marsh - "IF YOU DESIGN FOR THE SMALLEST, LEAST POWERFUL DEVICE FIRST, THEN YOU WILL FOCUS ON CONTENT, AND YOUR CORE FUNCTIONALITY"

There was a paradigm shift...decline in desktop -> rise in mobile.

--- Huge range of devices (different screen sizes, and resoluctions)

EVERYTHING WE BUILD NEEDS TO BE VIEWABLE ACROSS ALL THESE DEVICES


DESKTOP COMPUTERS --- usually shared within a household environment ----- used by multiple people ---- replaced every 4-5 years

MOBILE | TABLET | WATCH ----- replaced under every 2 years....so many devices are available - essentially 1 or 2 per person in the household.

**** THIS SHIFT HAS CHANGED THE INTERNET TRAFFIC ****

The simple guideline is whatever you are doing, do mobile first! --- though most people are still designing with a desktop first mindset


RESPONSIVE IS NOT MOBILE FIRST

--- Responsive Retrofitting: Taking what you already have and making it WORK on mobile.


MOBILE IS THE NEW DESKTOP

--- Can be time consuming, but it gives a much better experience across all of your devices.

--- It addresses constraints on mobile devices like SIZE and BANDWIDTH.

--- Planning for the future and growth.

Things to remember

  • Desktop sales have been declining since 2011
  • You don't see people with their heads in laptops
  • START SMALL & SCALE UP
  • -------- This ensures you are really concentrating on your content and functionality to make sure things are going to work well on those small devices
  • -------- Things tend to grow rather well
  • MOBILE IS NOT A FAD
  • Don't listen to what people have to say --- observe what they're doing and find a way to fill their needs as simply as possible.
